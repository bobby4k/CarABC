# AGENTS.md

## 目标
- 本仓库用于生成面向喜欢汽车的小男孩的 100 天学习卡片。
- 每天一张 A5 卡片，每张 A4 纸排版两天内容。
- 每日卡片固定包含：一张图片、一句汉字句子、一句英文句子、一个互动小任务。
- OpenCode 负责主题策划与内容规划，Python 脚本负责图片生成与 PDF 产出。

## 当前目录约定
- `AGENTS.md`：代理执行规则。
- `config.yaml`：阶段、模型、路径等配置。
- `generate_pdf.py`：Python 主脚本。
- `output/themes.yaml`：OpenCode 生成的结构化主题与卡片数据。
- `requirements.txt`：Python 依赖。
- `.env`：环境变量，例如 API Key。
- `hanzi/`：汉字字库目录。
- `output/`：日志与 PDF 输出目录。
- `cards/`：按天存放生成图片。

## 语言要求

- 新写入的文档、说明、规则优先使用中文。
- 代码标识符、命令、路径、环境变量保持常见英文形式。
- 若无明确要求，不切换到英文回复。

## 工作流分工
- OpenCode 先通过 `/plan` 与 `/build` 生成 100 天主题规划与 `output/themes.yaml`。
- 主题规划需保证多样性、不重复、符合阶段递进。
- Python 脚本读取 `output/themes.yaml` 与配置，执行图片生成、日志记录与 PDF 合成。
- 图片生成失败时，流程不能整体中断。

## 核心业务规则
- 每日主题不得重复。
- 主题可分为：汽车通用知识、经典汽车车型、经典玩具车模、历年销量榜车型、玩具品牌介绍等。
- 仅当主题本身是玩具品牌介绍时，正文允许出现品牌名。
- 非品牌主题中不得出现多美卡、拓易、Mini GT、风火轮等车模品牌名。
- 内容需适合 4-7 岁儿童，表达简单、直接、友好。

## 每日卡片字段
- `day`：天数，从 1 开始连续递增。
- `stage`：当前阶段名称，例如启蒙、基础、拓展。
- `theme`：主题名称，简短清晰。
- `theme_type`：`knowledge`、`car_model` 或 `brand`。
- `cn_sentence`：一句完整通顺的中文句子。
- `cn_pinyin_marks`：中文句子中重点词的拼音标注信息。
- `en_sentence`：一句零基础水平的英文句子。
- `en_word_notes`：重点英文词的中文释义。
- `image_style`：图片风格指示。
- `task`：简短互动任务。
- `image_prompt`：图片提示词。
- `image_path`：图片输出路径。

## YAML 数据结构规则
- 唯一主题输入源是 `output/themes.yaml`。
- 顶层建议使用 `days` 列表。
- 每个元素代表一天的完整卡片数据，不再使用 `themes.txt`。
- `day` 必须从 1 到目标天数连续编号，不能缺失或重复。
- `theme` 必须全局唯一。
- `cn_pinyin_marks` 与 `en_word_notes` 都应使用列表结构，便于后续排版。
- `image_prompt` 必须在 YAML 中直接给出，Python 直接读取，不依赖二次猜测。
- 参考结构如下：

```yaml
days:
  - day: 1
    stage: 启蒙
    theme: 小汽车
    theme_type: knowledge
    image_style: 吉卜力风格，有一定参考背景
    image_prompt: 一辆红色小汽车，儿童启蒙插画，吉卜力风格，主体明确，有一定参考背景，儿童友好，无文字
    cn_sentence: 小汽车开来了。
    cn_pinyin_marks:
      - word: 小汽车
        pinyin: xiao3 qi4 che1
    en_sentence: The car is coming.
    en_word_notes:
      - word: car
        note: 小汽车
    task: 找一找图里的车轮。
```

## 汉字字库规则
- 字库种子来自 `hanzi/qimeng.txt`、`hanzi/jichu.txt`、`hanzi/tuozhan.txt`。
- 配置文件中可为三个阶段追加其他常用字。
- 每个阶段字库采用累计规则，而不是互斥规则。
- 启蒙阶段使用启蒙字库。
- 基础阶段使用启蒙 + 基础字库。
- 拓展阶段使用启蒙 + 基础 + 拓展字库。
- 根据当前天数，从 `config.yaml` 的 `stages` 中确定当前阶段。
- 读取所有指定字库文件时，按空格分隔取字。
- 合并后必须去重，得到当前阶段可用汉字集。
- 文案生成应尽量优先使用当前阶段可用汉字集。

## 阶段规则
- 默认阶段范围由 `config.yaml` 控制，不要把天数写死在代码里。
- 典型配置为：`1-40` 启蒙、`41-70` 基础、`71-100` 拓展。
- 若用户修改阶段范围，脚本与内容逻辑都应跟随配置变化。
- 代理在生成主题或内容时，必须考虑当前阶段的可用字与认知难度。

## 主题规划规则
- 100 天主题要覆盖汽车认知、交通场景、车辆功能、工程车、救援车、零部件、规则意识、汽车品牌、经典车型等方向。
- 主题需要有变化，避免连续多天只是同类车辆换颜色。
- 启蒙阶段前段可先放通用认知主题，启蒙阶段后段就应开始适当加入经典车型。
- 小朋友通常会更早对实车、经典车型、经典工程车感兴趣，规划时可适当前置这类主题。
- 具体车型主题可以写“丰田埃尔法”、“AE86”等车型名。
- 经典工程车主题可写“压路机”“搅拌车”“吊车”等。
- 通用知识主题可写“车轮”“红绿灯”“救护车”“挖掘机”等。
- 品牌介绍主题可写多美卡、拓易、Mini GT、风火轮等。
- 非品牌主题不得夹带品牌露出。

## 中文句子规则
- 每天只写一句中文句子。
- 句子必须完整、通顺、贴合图片内容。
- 句子长度应短，便于 4-6 岁儿童跟读。
- 重点词语需要标注拼音位置，但不要整句全量注音。
- 尽量让句子与当前阶段字库匹配。
- 若必须使用阶段外常用字，应尽量少，并优先通过配置补充。

## 英文句子规则
- 每天只写一句英文句子。
- 英文句子与中文句意一致即可，不要求逐字直译。
- 难度保持零基础水平，优先使用短句型。
- 重点英文词需要标注中文意思。
- 避免复杂时态、从句和抽象表达。

## 互动任务规则
- 每天只提供一个任务。
- 任务要简短、可操作、与主题直接相关。
- 任务应适合亲子互动，1 分钟左右可完成。
- 优先使用观察、指出、数数、找颜色、说部件、模仿动作等形式。
- 不依赖额外教具为默认原则。

## 图片风格规则
- 品牌相关或具体车型主题：优先选择 `brand_names` 已出品的玩具车模型或所对应的实车，采用写实风，背景简洁。
- 交通规则类、安全类主题：采用吉卜力风格，有一定参考背景。
- 若主题同时涉及品牌与规则教育，以“是否属于规则/安全教育”为优先判断条件，优先使用吉卜力风格，有一定参考背景。
- 其他汽车或交通知识主题：默认使用吉卜力风格，有一定参考背景。
- 图片主体要明确，避免杂乱元素。
- 画面需儿童友好，不包含事故、血腥、惊吓内容。

## 图片生成规则
- Python 优先直接读取 `output/themes.yaml` 中的 `image_prompt`。
- 图片平台优先读取 `config.yaml` 中的模型配置。
- 支持通义万相或其他大模型平台，但接口细节必须通过配置控制。
- 生成图片前应先判断主题是否属于规则/安全教育，再判断是否属于品牌或具体车型。
- 品牌或车型提示词中应优先体现玩具车常见出品车型或其对应实车、写实风、主体明确、背景简洁。
- 规则或安全主题提示词中应优先体现儿童启蒙插画、动作清晰、吉卜力风格、有一定参考背景。
- 图片保存路径固定为 ``{image_dir}/day{xxx}/image.jpg``。
- 若图片生成失败，记录错误并继续执行后续天数。
- PDF 生成时允许该位置留白或使用占位图。

## 主题输入规则
- `output/themes.yaml` 是唯一主题输入文件。
- OpenCode 输出的数据应已包含主题类型、图片风格、图片提示词、中英文句子与互动任务。
- 脚本启动时至少校验一次 `day` 连续性、主题去重、字段完整性。
- 如果天数不足 `rules.total_days`，应报错并停止最终生成。
- 若 YAML 字段结构有变化，必须同步更新 `AGENTS.md`、`config.yaml` 与脚本读取逻辑。

## 配置文件规则
- `config.yaml` 是唯一配置入口。
- 阶段范围、字库文件、图片模型、输出路径都应从配置读取。
- API Key 不应写入 `config.yaml`，应通过环境变量读取。
- 环境变量名默认使用 `DASHSCOPE_API_KEY`。
- 用户可自行修改 `config.yaml`，代理不应假设其内容固定不变。

## 日志规则
- 生成日志写入 `output/generated_log.log`。
- 每天至少记录：天数、主题、阶段、图片状态、输出路径、错误信息。
- 日志要支持后续排查与重跑。
- 对可恢复错误使用记录并继续的策略。

## PDF 规则
- 最终输出文件默认是 `output/car_learning.pdf`。
- 页面尺寸默认 A4。
- 每页排版两张 A5 卡片。
- 缺失图片时，PDF 仍应可生成。
- 中文字体路径从配置读取，例如 `fonts/simsun.ttf`。
